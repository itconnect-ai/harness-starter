# 페이지 수정 후 체크리스트

페이지 콘텐츠(헤드라인, 문구, 가격, 수치 등)를 변경한 후 확인해야 할 항목입니다.
자동화할 수 없는 의미론적 일관성 검증이므로 사람이 직접 확인합니다.

## SEO 메타데이터

- [ ] `title`이 변경된 페이지 헤드라인과 일치하는가
- [ ] `description`이 변경된 콘텐츠를 반영하는가 (120자 내외)
- [ ] `canonical` URL이 올바른가
- [ ] `openGraph.title`, `openGraph.description`이 메타데이터와 동일한가

## AEO 구조화 데이터 (JSON-LD)

- [ ] FAQ 콘텐츠를 변경했다면 `FAQPage` 스키마의 질문-답변도 동일하게 수정했는가
- [ ] 서비스 설명을 변경했다면 `Service` 스키마의 description/OfferCatalog도 수정했는가
- [ ] 프로세스 단계를 변경했다면 `HowTo` 스키마도 수정했는가
- [ ] `Organization` 스키마의 정보(founder, contactPoint 등)가 최신인가

## GEO 인용 유도 문장

- [ ] 수치/통계가 변경됐다면 본문의 인용 유도 문장도 갱신했는가
  - 예: "4~6주" → "2~4주"로 바뀌면 메타데이터, 스키마, 본문 팩트 문장 모두 갱신
- [ ] 추상적 표현 대신 구체적 수치가 포함되어 있는가
- [ ] 비교 문장의 기준과 수치가 현재 사실과 일치하는가

## OG 이미지

- [ ] 페이지 핵심 메시지가 바뀌었다면 OG 이미지 텍스트도 갱신했는가
- [ ] 소셜 공유 미리보기가 변경된 콘텐츠를 반영하는가

## 검증

- [ ] Google Rich Results Test 통과 (구조화 데이터)
- [ ] Meta OG 디버거 / 카카오 OG 디버거로 미리보기 확인
- [ ] Lighthouse SEO 점수 90점 이상

---

## LLM에게 전달할 프롬프트

페이지 수정 후 아래 프롬프트를 LLM에게 전달하면 위 항목을 자동으로 처리합니다:

```
방금 수정한 페이지의 SEO/AEO/GEO 일관성을 확인하고 수정해줘.

docs/agents/seo-rules.md를 읽고 아래를 확인:
1. metadata (title, description, openGraph)가 변경된 콘텐츠와 일치하는지
2. JSON-LD 스키마 (FAQPage, Service, HowTo 등)가 본문과 동일한지
3. GEO 인용 유도 문장의 수치/통계가 현재 사실과 맞는지
4. OG 이미지 텍스트가 페이지 메시지와 일치하는지

불일치하는 항목은 직접 수정해줘.
```
